# PACScan 결과 기록

이 폴더는 PACScan/NeuroLens Streamlit 서비스의 개발 과정, 전처리 검증, 설계 결정 및 실행 결과를 재현 가능한 형태로 보관한다.

## 기록 원칙

모든 기록은 다음 질문에 답해야 한다.

1. 왜 변경하거나 실험했는가?
2. 무엇을 변경하거나 실행했는가?
3. 어떤 입력과 환경을 사용했는가?
4. 어떻게 검증했는가?
5. 어떤 결과가 나왔는가?
6. 문제와 한계는 무엇이었는가?
7. 무엇을 최종 선택했는가?
8. 다음 작업은 무엇인가?

## 파일 구성

| 파일 | 목적 |
|---|---|
| `01_개발_결과_기록.md` | Streamlit UI와 기능이 어떻게 변경되었는지 시간순으로 기록 |
| `02_설계_의사결정.md` | 원버튼, 결과 탭, N4 제외 등 중요한 결정과 근거 기록 |
| `03_ref21order_v1_전처리_검증.md` | 실제 전처리 산출물의 파일 수, 파라미터, 불일치 및 시연 활용 조건 기록 |
| `04_실험_기록_양식.md` | 이후 실험마다 복사해 사용하는 상세 기록 양식 |
| `05_분석_실행_로그_명세.md` | 실제 서비스 실행 시 저장할 구조화 로그 필드 정의 |
| `analysis_run_template.csv` | 실행 결과를 표 형태로 축적하기 위한 CSV 헤더 |

## 개인정보 원칙

- 환자 이름, 생년월일 및 원본 Patient ID를 공개 저장소에 기록하지 않는다.
- 환자 식별이 필요하면 별도 salt를 적용한 해시 ID만 기록한다.
- 실제 DICOM 및 의료영상 파일은 GitHub에 커밋하지 않는다.
- 경로, 로그 및 오류 메시지에 개인정보가 포함되지 않았는지 확인한다.
