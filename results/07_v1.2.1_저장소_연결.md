# PACScan v1.2.1 저장소 연결 기록

- 변경일: 2026-07-23 KST
- 목적: UI 저장소와 연구 코드 저장소를 분리한 상태로 안정적으로 연결
- PACScan 역할: DICOM 업로드, 실행 제어, 결과 표시, 실행 로그 저장
- BRAINTENSOR 역할: `ref21order_v1` 실제 전처리와 향후 모델 추론

PACScan은 다음 우선순위로 전처리 코드를 찾는다.

1. `PACSCAN_BRAINTENSOR_SCRIPT` 환경변수
2. PACScan과 같은 상위 폴더의 `BRAINTENSOR` 저장소
3. 기존 개발 PC 기본 경로 `E:\해커톤\BRAINTENSOR`

연결된 BRAINTENSOR의 Git commit은 매 실행의 `run.json`과 `run_index.csv`에 기록한다.
새 환경에서는 `scripts/setup_local.ps1`로 두 저장소를 나란히 복제하고 로컬 의존성을
설치할 수 있다. 의료영상과 환자별 결과는 PACScan Git 저장소에 커밋하지 않는다.
