# PACScan v1.2.0 로컬 실제 전처리 연결 기록

## 변경 일시

- 2026-07-23 (KST)
- 앱 버전: `1.2.0`
- 연결 파이프라인: `BRAINTENSOR ref21order_v1`

## 목적

Streamlit에서 환자 한 명의 원본 T2 DICOM 폴더를 선택하면 GitHub에서 관리하는
BRAINTENSOR 전처리 코드를 로컬 PC에서 직접 실행하고, 완성된 NIfTI와 처리 화면을
PACScan에 표시한다.

## 실제 실행 순서

1. DICOM 폴더 검사 및 T2 시리즈 자동 선택
2. DICOM을 3D NIfTI로 변환
3. FSL BET 뇌 추출 (`frac=0.5`)
4. ANTsPy `SyN`과 PD25 T1 MPRAGE 1 mm 아틀라스를 이용한 정합
5. 비영역 voxel 기준 Min-Max 정규화
6. N4 bias correction 제외 (`false`)
7. `56×56×56` 선형 리사이즈
8. 결과 NIfTI, 원본/전처리 3면 영상 및 QC 정보를 Streamlit에 표시

## 실행환경 선택

- 로컬에서 BRAINTENSOR 스크립트, PD25 아틀라스, FSL BET, antspyx가 모두 확인되면
  `local_full` 모드로 실제 파이프라인을 실행한다.
- GitHub/Streamlit Cloud처럼 전체 연구환경이 없는 곳에서는 기존의 배포 가능한
  방향 표준화·Min-Max·56³ 경량 파이프라인만 실행하며 화면에 경량 모드임을 명시한다.
- 로컬에 BRAINTENSOR 코드가 있지만 필수 도구가 누락된 경우 경량 결과로 몰래
  대체하지 않고 실행을 중단하여 누락된 도구를 표시한다.

## 자동 기록 구조

각 로컬 실행은 `results/runs/<run_id>/`에 분리된다.

```text
results/runs/<run_id>/
├── run.json          # 시간, 앱/코드 버전, 파라미터, 성공/실패
├── steps.jsonl       # 단계별 시작·종료·소요 시간·오류
├── artifacts.json    # 결과 파일명, 크기, SHA-256, shape
├── error.json        # 실패한 경우 오류와 traceback
├── input_dicom/      # 실행에 사용한 선택 시리즈(로컬 전용)
└── artifacts/        # 최종 NIfTI와 중간 산출물(로컬 전용)
```

전체 실행 검색용 색인은 `results/runs/run_index.csv`에 누적한다. 환자 식별자는 원문
대신 SHA-256 일부만 `run.json`에 기록한다. DICOM과 NIfTI 및 환자별 실행 폴더는
`.gitignore`로 보호하여 GitHub에 올라가지 않게 한다.

## 파일별 역할

- `local_pipeline.py`: 환경검사, GitHub 코드 동적 로딩, 실제 실행, 단계 계측, 로그 저장
- `app.py`: 단일 분석 버튼에서 로컬 전체/클라우드 경량 모드를 선택하고 결과 표시
- `results/version_index.csv`: 버전별 변경 사항의 누적 인덱스
- `results/06_v1.2.0_로컬_전처리_연결.md`: 이번 변경의 상세 근거와 결과
