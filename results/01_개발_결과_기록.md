# PACScan Streamlit 개발 결과 기록

## 프로젝트 목표

환자 한 명의 T2 MRI DICOM 폴더를 입력받아 전처리 상태, 3방향 MRI, AI 분석 데모, XAI 근거 및 임상 보고서를 한 화면에서 확인할 수 있는 의료진용 Streamlit 프로토타입을 구현한다.

---

## DEV-001 — 초기 의료 대시보드 구현

### 목적

제공된 NeuroLens 대시보드 시안을 Streamlit으로 재현하고 모델 연결 전에도 전체 서비스 흐름을 확인한다.

### 구현

- Deep Navy 기반 의료용 다크 테마
- 환자정보, MRI 뷰어, 진단 확률, 모델 정보, 판독 소견 카드
- 예시 파일 다운로드 및 업로드
- 향후 모델 연결을 위한 추론 어댑터 분리

### 결과

- Streamlit 서버 HTTP 200 응답 확인
- Python 구문 검사 통과
- Streamlit AppTest 예외 0건

### 발견된 문제

- 초기 버전은 전체 대시보드 캡처를 MRI처럼 표시하여 실제 입력·결과 구분이 부족했다.
- 업로드 전, 처리 중, 결과 완료 상태가 명확하게 분리되지 않았다.

### 결정

업로드 상태와 분석 상태를 분리하고 실제 MRI 단면을 각 영역에 표시하도록 재구성한다.

### 관련 커밋

- `636fecb` — Add PACScan Streamlit prototype

## DEV-002 — 반응형 레이아웃 수정

### 목적

좁은 화면에서 왼쪽 메뉴와 환자정보가 한 글자씩 세로로 줄바꿈되는 문제를 해결한다.

### 원인

Streamlit의 3열 레이아웃이 중간 화면 폭에서도 유지되면서 사이드 열의 실제 너비가 텍스트 최소 너비보다 작아졌다.

### 구현

- 1,100px 이하에서 Streamlit column을 세로 배치
- 왼쪽 세로 메뉴를 가로 스크롤 메뉴로 전환
- 환자정보 및 결과 패널을 전체 너비로 표시
- 650px 이하에서 MRI 보조 단면을 모바일 배치로 변경

### 결과

- 좁은 화면에서 글자가 한 글자씩 분리되는 현상 해소
- 데스크톱에서는 기존 고밀도 의료 대시보드 유지

### 관련 커밋

- `2d85f7c` — Fix responsive layout for narrow screens

## DEV-004 — 원버튼 분석 워크플로

### 목적

전처리 단계마다 별도 버튼을 누르는 불편을 제거하고 한 번의 실행으로 전체 흐름을 시작한다.

### 변경 전

업로드, 전처리, 분석 및 보고서 생성 단계가 각각 별도 조작처럼 보였다.

### 변경 후

사용자가 누르는 주요 실행 버튼은 `분석 시작` 하나로 제한했다.

```text
입력 검사 → 변환 → 전처리 → QC → AI 결과 화면 준비
```

완료 후에는 실행 버튼 대신 결과 탐색 탭을 사용한다.

- 원본 MRI
- 전처리 결과
- AI 분석(데모)
- XAI 보고서

### 결과

서비스 시연 흐름이 단순해지고 사용자가 다음에 눌러야 할 버튼을 고민하지 않아도 된다.

### 관련 커밋

- `cde9223` — Add one-click NIfTI preprocessing workflow

## DEV-007 — DICOM 환자 폴더 업로드

### 목적

단일 NIfTI가 아니라 환자 한 명의 여러 DICOM 슬라이스가 들어 있는 폴더를 직접 입력받는다.

### 구현

- Streamlit directory uploader 사용
- 모든 하위 폴더 파일 수집
- pydicom으로 DICOM 헤더 검사
- `SeriesInstanceUID` 기준 시리즈 분류
- `SeriesDescription` 및 `ProtocolName`에서 T2 탐색
- Localizer, Scout 및 FLAIR 선택 감점
- T2 표기가 없으면 최대 슬라이스 시리즈 선택
- dicom2nifti로 3D NIfTI 변환

### 검증 입력

- 환자 ID: `PACSCAN-DEMO-001`
- DICOM: 32개
- SeriesDescription: `T2 AXIAL`
- ProtocolName: `T2_TSE_AX`
- 원본 volume: `96×96×32`

### 검증 결과

- DICOM 인식: 성공
- T2 자동 선택: 성공
- DICOM→NIfTI 변환: 성공
- 변환 방향: LAS
- 56³ 변환: 성공
- 3방향 미리보기: 성공

### 관련 커밋

- `6e1099f` — Support patient DICOM folder preprocessing
